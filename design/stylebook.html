Rules
Heading weight is Lexend 600 — not 700
700 makes headings appear chunky against the dark surface. 600 hits the ideal balance of authority and elegance for financial data interfaces.
All monetary values use tabular numerals
Add font-variant-numeric: tabular-nums to any element displaying currency, percentages, or counts. This prevents layout shift as numbers change.
Use uppercase + letter-spacing for micro-labels only
Labels like "TOTAL INCOME" or "APR 2026" use Lexend 500, 11–12px, letter-spacing 0.07–0.10em. Never apply this style to body text or values.

Always pair color with a symbol or label
Never rely on color alone. Positive values must show "+" or "↑". Negative values must show "−" or "↓". This satisfies WCAG color-independence requirements and supports colorblind users.
Warning vs Danger threshold
Use Amber (#F59E0B) for 80–99% budget used (warning, not urgent). Switch to Red (#EF4444) only when the limit is exceeded. Never use red for anything under 100%.

Rules
Scrim: rgba(5, 10, 20, 0.75) + backdrop-filter: blur(2px)
Strong enough to isolate the modal but preserves spatial context. Never use pure black. The slight blue tint in the scrim aligns with the surface colour temperature.
Modal width: 380–480px. Never full-width on desktop.
Small modals (confirm/alert): 380px. Form modals: 440px. Complex multi-field: 480px. Wider than 480px becomes a side sheet instead.
Destructive modals use a red-tinted top border and red title
The header border-bottom changes to rgba(239,68,68,0.2) and the title uses --negative. The primary action button uses btn-danger, never btn-primary.
Border-radius: 20px (--r-xl) Animation: scale(0.96)→scale(1) + opacity 0→1 Duration: 220ms ease-out Header padding: 22px 24px Body padding: 22px 24px Footer padding: 16px 24px

Rules
Grid lines: rgba(255,255,255,0.04) — barely visible
Grid lines orient the eye without competing with data lines. Never use a visible colour — they should be invisible until the user looks for them.
Current period bars get a subtle glow; future periods use Surface 3 fill
Apply box-shadow: 0 0 8px rgba(color, 0.4) to the active month only. Future/empty months use --surface-3 fills to indicate no data yet.
Tooltips always show: label, date, income value, spend value
Background: Surface 3 (#1E2A42). Border: --border-strong. Font: Source Sans 3, 11–12px. Position above the cursor, never overlapping the data point.
Income line/bar: --positive (#10B981) Spending line/bar: --negative (#F87171) Net / Remaining: --blue (#3B82F6) Stroke width: 2px Bar radius: 3px top Area opacity: 0.15–0.25
